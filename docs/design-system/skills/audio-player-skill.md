# Audio Player Skill Draft

用途：当 AI 需要为少年博文页面新增或调整音频播放能力时，按本工作流执行。  
依赖：

- `../mvp-design-system-v0.1.md`
- `../modules/audio-player.md`

## 1. 触发条件

当需求包含以下任一表达时，使用本 skill：

- 听全文。
- 速听今日大事。
- 音频播放条。
- 跟读。
- 朗读新闻。
- 迷你播放器。
- 播放、暂停、进度、时长。

## 2. 先判断音频形态

AI 必须先判断使用哪种形态：

- `bar`：完整横向播放条，默认用于首页和详情页。
- `inline`：行内听句子或跟读按钮。
- `mini`：迷你播放器，适合滚动或跨页面播放。
- `sticky`：页面内吸附播放器，适合长文阅读。

默认选择：

- 首页速听使用 `bar`。
- 详情页听全文使用 `bar`。
- 中英对照跟读使用 `inline`。
- 长文滚动播放后续可升级 `mini` 或 `sticky`。

## 3. 必须输出的状态

必须覆盖：

- 未播放 `idle`。
- 加载中 `loading`。
- 播放中 `playing`。
- 暂停 `paused`。
- 播放完成 `ended`。
- 加载失败 `error`。
- 无音频 `disabled` 或 fallback。

## 4. 设计规则

必须遵守：

- 读取 `audio-player.md` 中的模块结构。
- 完整播放条默认宽度跟随主内容 `350px`。
- 完整播放条默认高度 `52px`。
- 背景使用 `Bg 背景/Bg_Audioplay 音频播放`。
- 同一页面同一时刻只突出一个主音频。
- 播放失败只影响音频模块，不影响正文阅读。

禁止：

- 在首页信息流连续放多个完整播放器。
- 让播放器遮挡评论输入、底部导航或正文。
- 只设计未播放态。
- 将行内跟读按钮设计成完整播放器。
- 每个页面各自发明播放控件。

## 5. 输出格式

AI 生成音频模块方案时，输出：

```md
## 音频模块接入方案

音频形态：
放置位置：
播放内容：
组件结构：
状态覆盖：
数据需求：
视觉约束：
交互说明：
失败与 fallback：
人工审核清单：
```

## 6. 接入模板

### 首页速听

```txt
HomePage
  HomeTopBar
  AudioPlayerBar(bar)
  NewsFeed
```

规则：

- 放在顶部问候区之后、信息流之前。
- 文案强调“速听”或“今日大事”。

### 详情页听全文

```txt
ArticleDetailPage
  ArticleHero
  AudioPlayerBar(bar)
  ImpactSummary
```

规则：

- 放在新闻标题和元信息之后。
- 不遮挡正文阅读。

### 中英对照跟读

```txt
BilingualModule
  ParagraphPair
    InlineListenButton(inline)
```

规则：

- 行内按钮只控制当前句段。
- 不展示完整播放条，避免重复和拥挤。

## 7. 审核清单

设计：

- 是否覆盖所有播放状态。
- 播放控件是否和页面层级匹配。
- 是否处理底部安全区和评论输入冲突。

产品：

- 音频内容类型是否明确：摘要、全文、句子、简报。
- 是否需要 transcript、跟读、下载。

工程：

- 是否有统一播放状态。
- 是否处理多个音频互斥。
- 是否保留失败重试和无音频 fallback。
